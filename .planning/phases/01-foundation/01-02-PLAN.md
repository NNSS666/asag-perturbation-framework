**`src/asag/splitters/protocol_b.py` — Within-question splitter (DATA-05):**
- Define `protocol_b_splits(answers: List[AnswerRecord], test_size: float = 0.2, random_state: int = 42) -> Dict[str, Tuple[List[str], List[str]]]`
- Group answers by `question_id` using `collections.defaultdict`
- For each question's answers:
  - Extract gold_labels for stratification
  - If the question has < 5 answers OR < 2 distinct gold_labels: fall back to simple positional split at `int(len * (1 - test_size))` (see Pitfall 5 in RESEARCH.md). Log which questions hit this fallback.
  - Otherwise: use `sklearn.model_selection.StratifiedShuffleSplit(n_splits=1, test_size=test_size, random_state=random_state)` to split answer indices, stratifying on gold_label
  - Collect train_answer_ids and test_answer_ids
- Return dict mapping question_id -> (train_answer_ids: List[str], test_answer_ids: List[str])
- Add logging: total questions split, number that used fallback split, overall train/test ratio
- Add `if __name__ == "__main__"` block that loads Beetle, runs Protocol B, and prints per-question split statistics

**Edge case handling (from RESEARCH.md Pitfall 5):**
- Questions with only 1 answer: put it in train (can't test on nothing)
- Questions with 2-4 answers: use simple split (stratification unreliable)
- Questions with all same label: use simple split (stratification impossible)

IMPORTANT: Pass `random_state` explicitly to StratifiedShuffleSplit for per-component seed isolation (INFR-01 / Pitfall 6 in RESEARCH.md).
